What is Kafka?

Apache Kafka is a publish-subscribe messaging system which lets you send messages between processes, applications, and servers.

Why Kafka?

Multiple source systems may communicate with multiple target systems. Intergration is difficult, hence Kafka comes as a middle layer.
Decoupling of Data Streams.
Real time instead of batch processing.

Topics

It is a particular stream of data similar to tables in databases.

Partitions

Each topics are split in partitions. Each message in partition is identified by an incremental id called offset.
Data is only kept for a limited time(Default is one week).
Once the data is written in partition it canâ€™t be changed(Immutability).

Broker

A Kafka cluster consists of one or more servers (Kafka brokers). Producers are processes that publish data (push messages) into Kafka topics within the broker. A consumer of topics pulls messages off a Kafka topic. Partitions are spread across these brokers.
If you are connected with one broker, you are connected with whole cluster. Messages are stored in any of the below brokers. Kafka takes care of storing and fetching the message from these brokers.

Topic replication should be greater than 1. That means if any of the broker is down data can be served from another replicated broker.
Each partition has one leader and multiple ISRs(in sync replicas). All reads and writes happens through leader.
Producers can choose to wait for acknowledgement of data writes. Acks=0, no wait; Acks=1, wait for Ack from leader; Acks=all, wait for leader and replicas.

Producer: Message Keys

If producers send keys with message, it guarantees that all the messages from the topic are part of same partition. This allows ordering of the data.

Consumer Group

Consumer can be grouped together for a given topic for maximising read throughput. Each consumer in a group read from mutually exclusive partitions which means if C3 in a consumer group is reading from partition P0, no other consumer from the group can read from P0 partition.

Zookeeper

Kafka needs Zookeeper to work. Zookeeper sends notification to the Kafka(in case of broker died, added, leader selected etc). It manages brokers, helps in performing leader election for the partition.
